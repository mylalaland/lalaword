앱 개요
앱 이름 LalaWord (라라워드)
한 줄 설명 영어 책을 찍으면 단어를 알려주는 아이용 영어 학습 앱
타겟 사용자 영어를 공부하는 한국 아이 (유치원~고등학생) + 부모
플랫폼 Web (PWA) — 모바일 브라우저 최적화, 추후 앱 스토어 래핑 가능
OCR 엔진 Google Gemini API (사용자 본인 API Key 입력, 무료 쿼터)
서버 최소화 — 로그인(인증)만 서버 사용, 나머지 모든 데이터는 기기(IndexedDB)에 저장
로그인 Google 로그인 + 카카오톡 로그인
과금 무료 (사용자가 자신의 Gemini API 무료 쿼터 사용)
핵심 컨셉: 아이가 영어 책을 10페이지 정도 촬영하면, AI가 텍스트를 인식하고 아이의 수준에 맞는 단어를 선별하여 뜻·발음·예문을 제공합니다. 부모가 난이도를 설정하고 학습 현황을 그래프로 확인할 수 있습니다.

3
영어 수준 등급 체계
설계 원칙: 한국에서 흔히 쓰는 영어 실력 등급 기준을 적용합니다. 부모가 아이의 현재 수준을 설정하면, 해당 수준 이상의 단어만 학습 대상으로 선별됩니다. "이 단어는 이미 아는 단어"를 자동으로 걸러줍니다.
등급 대상 표시 이름 단어 범위 설명
Lv.1 유치원 🌱 씨앗 기초 200단어 apple, dog, cat 등 일상 기초 단어 모두 표시
Lv.2 초등 저학년 (1~3) 🌿 새싹 초등 필수 800단어 교과서 기초 단어, 간단한 형용사·동사
Lv.3 초등 고학년 (4~6) 🌳 나무 초등 심화 1500단어 초등 교과서 전범위, 기본 숙어
Lv.4 중학생 ⭐ 별 중등 필수 2500단어 중학교 교과서 단어, 기본 문법 용어
Lv.5 고등학생 🚀 로켓 수능 필수 4000단어 수능 빈출 어휘, 학술적 표현
Lv.6 고급 / 성인 👑 왕관 고급 단어만 토익/토플 수준, 전문 어휘만 표시
수준 동작 방식
  • Lv.1 (씨앗): 스캔한 텍스트의 모든 단어를 표시 — 가장 쉬운 단어까지 전부 알려줌
  • Lv.3 (나무): 초등 저학년 수준 단어는 이미 안다고 판단 → Lv.3 이상 단어만 표시
  • Lv.6 (왕관): 쉬운 단어는 전부 스킵 → 고급/전문 단어만 표시
  • 설정 고정: 부모가 PIN으로 잠금 → 아이가 수준 변경 불가

데이터 설계 (IndexedDB)
설계 원칙: 모든 학습 데이터는 사용자 기기의 IndexedDB에 저장됩니다. 서버에 저장되는 데이터는 없습니다. 데이터 내보내기/가져오기 기능으로 기기 간 이동을 지원합니다.
📦 IndexedDB 스토어 목록
스토어명 용도 주요 필드
settings 앱 설정 level, translationMode, parentPIN, levelLocked, dailyGoal, apiKey
scans 스캔 기록 id, date, images[], extractedText, wordCount, bookTitle
words 단어장 id, word, meaning, pos, pronunciation, level, examples[], favorite, mastered
reviews 간격반복 데이터 wordId, nextReview, interval, ease, repetitions
dailyStats 일별 통계 date, wordsLearned, wordsReviewed, scanCount, studyMinutes
quizResults 퀴즈 결과 id, date, quizType, totalQuestions, correctCount, words[]
9
API 및 OCR 설계
Gemini API 호출 전략
  • 모델: Gemini 2.0 Flash (무료 쿼터 제공, Vision 지원)
  • API Key: 사용자가 직접 Google AI Studio에서 발급받아 입력
  • 무료 쿼터: 분당 15 요청 / 일일 1500 요청 (변동 가능)
  • 보안: API Key는 IndexedDB에 암호화 저장, 서버로 전송 안 함
OCR + 단어 추출 프롬프트 설계

1단계: 이미지 → 텍스트 추출 (Vision)

이 이미지는 영어 책의 한 페이지입니다. 이미지에서 모든 영어 텍스트를 정확하게 추출해주세요. 문단 구조와 줄바꿈을 유지해주세요. JSON 형식으로 응답해주세요: { "text": "추출된 전체 텍스트", "paragraphs": ["문단1", "문단2"] }

2단계: 텍스트 → 단어 분석

다음 영어 텍스트에서 단어를 분석해주세요. 사용자의 영어 수준: {level} (예: 초등 고학년) {level} 수준에서 학습이 필요한 단어만 선별해주세요. 각 단어에 대해 JSON으로 응답: { "words": [ { "word": "단어", "meaning": "한국어 뜻", "pos": "품사", "pronunciation": "발음기호", "level": 3, "example": "원문에서의 예문", "exampleMeaning": "예문 해석" } ] }

3단계: 문장 해석 (선택)

다음 영어 문장들을 한국어로 해석해주세요. 해석 모드: {mode} (직역 / 의역) 각 문장별 JSON 응답: { "translations": [ { "original": "영어 문장", "translation": "한국어 해석", "grammarPoints": ["문법 포인트"] } ] }
API 호출 최적화
  • 배치 처리: 10장의 이미지를 하나의 요청으로 처리 (멀티 이미지 Vision)
  • 캐싱: 이미 분석한 텍스트의 해시를 저장 → 중복 요청 방지
  • Rate Limiting: 분당 최대 10 요청으로 제한 (무료 쿼터 보호)
  • 에러 처리: 쿼터 초과 시 친절한 안내 메시지 표시

SM-2 간격반복 알고리즘
SM-2 알고리즘: 1987년 P. Wozniak이 개발한 간격반복 알고리즘. Anki에서 사용하는 것과 동일한 원리입니다. 단어의 기억 강도에 따라 복습 간격을 자동 조절합니다.
📐 알고리즘 공식
변수 설명 초기값
repetitions 연속 정답 횟수 0
interval 다음 복습까지 일수 1
ease 용이도 계수 (최소 1.3) 2.5
⚙️ 응답 품질 등급 (아이 친화적 UI)
등급 UI 표시 q 값 동작
❌ 모르겠어요 빨간 버튼 0~2 repetitions=0, interval=1 (내일 다시)
🤔 헷갈려요 노란 버튼 3 repetitions=0, interval=1 (기억 약함)
😊 생각났어요 초록 버튼 4 interval 증가 (정상 학습)
⭐ 완벽해요! 파란 버튼 5 interval 크게 증가 (숙달)
🔄 간격 계산 로직
function calculateSM2(word, quality) { // quality: 0~5 (사용자 응답 품질) let { repetitions, interval, ease } = word; if (quality >= 3) { // 정답 if (repetitions === 0) interval = 1; // 첫 정답: 1일 후 else if (repetitions === 1) interval = 6; // 두 번째 정답: 6일 후 else interval = Math.round(interval * ease); // 이후: interval × ease repetitions++; } else { // 오답: 처음부터 다시 repetitions = 0; interval = 1; } // 용이도 업데이트 ease = ease + (0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02)); if (ease < 1.3) ease = 1.3; return { repetitions, interval, ease, nextReview: addDays(new Date(), interval) }; }

질의사항 및 결정 필요 항목
질의 1
플랫폼 선택: Web(PWA) vs React Native(Expo)?
현재 Web(PWA)로 계획했습니다. 카메라 접근, 로컬 저장 모두 웹에서 가능하지만, 네이티브 앱 느낌을 원하시면 Expo로 변경할 수 있습니다. 이전 ReadnQuiz 프로젝트처럼 Expo로 가시겠습니까?
질의 2
영어 수준 등급의 세부 조정
현재 6단계로 설계했습니다. 한국에서 흔히 쓰는 등급으로는 TOEIC 점수대, 영어 인증시험 급수(토셀, 펠스) 등이 있는데, 아이 대상이므로 학년 기반이 맞을까요? 아니면 실제 시험 등급(토셀 Basic → Advanced)을 병기할까요?
질의 3
게스트 모드 지원 여부
로그인 없이 제한된 기능(예: 하루 3회 스캔)을 사용할 수 있는 게스트 모드를 넣을까요? API Key는 어차피 사용자가 직접 넣어야 하므로 게스트도 가능합니다.
질의 4
단어 발음: TTS만 사용 vs 발음기호도 표시?
Web Speech API의 TTS로 발음을 들을 수 있습니다. 추가로 IPA 발음기호도 표시할까요? (예: /əˈpɪə.rəns/)
질의 5
디자인 컨셉 세부
현재 코랄(주황빛 빨강) + 틸(청록) + 아이보리 배경의 따뜻한 톤으로 계획했습니다. 다른 컬러 방향이 있으시면 알려주세요. 예를 들어:
  • A) 현재 안: 코랄 + 틸 (따뜻하고 친근)
  • B) 민트 + 피치 (더 부드럽고 아기자기)
  • C) 네이비 + 골드 (신뢰감 있고 차분)
질의 6
책 제목/시리즈 관리 기능 필요 여부
스캔할 때 "어떤 책인지" 입력받아서 책별로 단어를 그룹핑할까요? 아니면 날짜별로만 관리하면 충분할까요?
질의 7
